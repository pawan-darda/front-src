Examples of entry in this field:
  • 2011.1 returns the versions that contain 2011.1 in the name
  • 2010 returns the versions that contain 2010 in the name in the name. Warning: Note that this is very likely a later version, as the date is in the name, and most 2011.1 version are built in 2010.
  • main:4 sub:6 -bn:200 returns the versions that are 4.6.x.y, with x not equals to 200 (note the minus sign)
  • sub:6 -nice:2010.2.3 returns the version that do not have a nice name equals to 2010.2.3 but do have a build number like x.6.n.m
  • comp:PRIME rel:2010.1.5 returns the version for the component PRIME in the release 2010.1.5
Versions look like 2011.1-4.7.513.0-20101111.
There is :
  • the nice name first (2011.1 here)
  • then main (4),
  • sub (7),
  • bn (513),
  • and abn (0) numbers,
  • then the build date (20101111)
Other filters than main, sub and nice are:
  • abn:NN (alternative build number, for special branches)
  • hf:NN for hotfixes,
  • p:NN for p-releases.
If you use the last:NN keyword (ex: last:20), the scenario filter will apply only to the last versions (in this case, the last 20). For instance, this is useful to filter the scenarios that have been failing in the last 20 versions. Otherwise, the scenario selectorwill apply to any version matching the version filter.
